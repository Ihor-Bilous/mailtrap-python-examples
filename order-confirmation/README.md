# Python E-commerce: Order Confirmation Email

A FastAPI application that receives Stripe `checkout.session.completed` webhooks and sends order confirmation emails via the [Mailtrap](https://mailtrap.io) template API — with signature verification, line item parsing, and duplicate event handling.

## How It Works

```
Stripe Checkout completes
        |
        v
  Webhook received
  POST /stripe/webhook
        |
        v
  Verify Stripe signature ── Invalid? → 400 reject
        |
        v
  Check for duplicate ── Already processed? → 200 skip
        |
        v
  Retrieve session + line items via Stripe API ── Failed? → 500 (Stripe retries)
        |
        v
  Parse order data (items, total, shipping)
        |
        v
  Send order confirmation email ── Failed? → 500 (Stripe retries)
  via Mailtrap template API
  (template_uuid + variables map)
        |
        v
  Store event ID in SQLite (24h dedup window)
        |
        v
  200 OK
```

## Features

- **Stripe Webhook Verification** — Validates webhook signatures using `stripe.Webhook.construct_event()` to reject tampered payloads
- **Order Data Extraction** — Retrieves the Checkout Session and fetches line items to get product names, quantities, and prices
- **Mailtrap Template API** — Sends emails using Mailtrap's template engine with a variables map (`order_id`, `items`, `total`, `shipping_address`)
- **Duplicate Handling** — Stores processed event IDs in SQLite for 24 hours to handle Stripe webhook retries gracefully
- **Graceful Error Handling** — Mail and Stripe API failures return `500` so Stripe retries delivery; invalid signatures return `400`; duplicate events return `200` and are skipped

## Prerequisites

- Python 3.12+
- [Mailtrap account](https://mailtrap.io) with a verified sending domain and an email template
- [Stripe account](https://dashboard.stripe.com) in test mode
- [Stripe CLI](https://stripe.com/docs/stripe-cli) for local webhook forwarding

## Setup

1. **Clone and install**

   ```bash
   git clone https://github.com/Ihor-Bilous/mailtrap-python-examples.git
   cd mailtrap-python-examples/order-confirmation
   pip install -r requirements.txt
   ```

2. **Configure environment**

   ```bash
   cp .env.example .env
   ```

3. **Set your API keys** in `.env`

   ```
   # Mailtrap — find API key at mailtrap.io → API Keys
   MAILTRAP_API_TOKEN=your_mailtrap_api_token

   # Mailtrap — template UUID from Email Templates → edit template → UUID field
   MAILTRAP_TEMPLATE_UUID=your_order_confirmation_template_uuid

   # Your verified sending domain in Mailtrap
   MAIL_FROM=orders@yourdomain.com
   MAIL_FROM_NAME=Your Store

   # Stripe — find secret key at Stripe Dashboard → Settings → Developers → Manage API keys
   STRIPE_SECRET_KEY=sk_test_your_stripe_secret_key

   # Stripe — generated by running `stripe listen` (see Stripe Test Mode Setup below)
   STRIPE_WEBHOOK_SECRET=whsec_your_webhook_signing_secret

   # Fallback recipient when Stripe session has no customer email (useful for testing)
   ORDER_RECIPIENT_EMAIL=your@email.com
   ```

4. **Run the app**

   ```bash
   uvicorn main:app --reload
   ```

## Stripe Test Mode Setup

### 1. Create a test product and price

In the [Stripe Dashboard](https://dashboard.stripe.com/test/products) (test mode):

1. Go to **Products** > **Add product**
2. Set a name (e.g., "Widget Pro") and price (e.g., $19.99)
3. Save — Stripe creates a Product and a Price automatically

### 2. Install and configure Stripe CLI

```bash
# Install (macOS)
brew install stripe/stripe-cli/stripe

# Login to your Stripe account
stripe login
```

### 3. Forward webhooks to your local app

```bash
stripe listen --forward-to localhost:8000/stripe/webhook
```

The CLI prints a webhook signing secret (`whsec_...`). Copy it to your `.env` as `STRIPE_WEBHOOK_SECRET`.

### 4. Trigger a test event

In a separate terminal:

```bash
stripe trigger checkout.session.completed
```

> **Note:** `stripe trigger` sends a synthetic event with a fixed customer email (`stripe@example.com`) and real line items from the test fixture. To receive the email at your own address, set `ORDER_RECIPIENT_EMAIL` in `.env` — this is used as a fallback when the Stripe session has no real customer email.

**For a full end-to-end test with real customer email, items, and shipping:**

1. Go to [Stripe Dashboard](https://dashboard.stripe.com) → **Product catalogue** → **Add product**
2. Go to **Payment Links** → create a payment link for your product
3. Enable **"Collect customer addresses"** → **Shipping and billing** so `shipping_address` is populated
4. Open the payment link and complete checkout with:
   - Your real email address
   - Test card `4242 4242 4242 4242`, any future expiry, any CVC
5. The webhook fires with real customer data and the order confirmation email is sent to your address

## Mailtrap Template Setup

1. Go to [Mailtrap](https://mailtrap.io) > **Sending** > **Email Templates**
2. Create a new template for order confirmation
3. Use these template variables in your design:

| Variable | Type | Example |
|---|---|---|
| `order_id` | string | `"cs_test_a1b2c3d4"` |
| `items` | array | `[{"name": "Widget Pro", "quantity": 2, "price": "USD 39.98"}]` |
| `total` | string | `"USD 49.99"` |
| `shipping_address` | object or null | `{"name": "Jane Doe", "line1": "123 Main St", "line2": "", "city": "Springfield", "state": "IL", "postal_code": "62701", "country": "US"}` |

4. Copy the template UUID (not the numeric ID) to your `.env` as `MAILTRAP_TEMPLATE_UUID`

## Key Integration Points

### Stripe Webhook Signature Verification

Every incoming webhook is verified before processing:

```python
event = stripe.Webhook.construct_event(
    payload, sig_header, settings.stripe_webhook_secret
)
```

Invalid or tampered payloads are rejected with a `400` response.

### Mailtrap Template API

Order confirmation emails are sent via Mailtrap's template engine:

```python
mail = mt.MailFromTemplate(
    sender=mt.Address(email=settings.mail_from, name=settings.mail_from_name),
    to=[mt.Address(email=order.customer_email)],
    template_uuid=settings.mailtrap_template_uuid,
    template_variables={
        "order_id": order.order_id,
        "items": [...],
        "total": order.total,
        "shipping_address": {...},
    },
)
mt.MailtrapClient(token=settings.mailtrap_api_token).send(mail)
```

### Duplicate Event Handling

Stripe may retry webhook delivery. The app stores each processed event ID in SQLite for 24 hours:

```python
if dedup.is_processed(event["id"]):
    return JSONResponse({"status": "already_processed"})

# ... process event ...

dedup.mark_processed(event["id"])
```

### Error Handling

| Scenario | HTTP response | Stripe retries? |
|---|---|---|
| Invalid signature | `400` | No |
| Malformed payload | `400` | No |
| Duplicate event | `200 {"status": "already_processed"}` | No |
| Stripe API error | `500` | Yes |
| Mailtrap send failure | `500` | Yes |
| Success | `200 {"status": "ok"}` | No |
